Information Architechture revamp for better discoverabilty of features & data organization

UX RESEARCH - UX DESIGN

Let's start with the problem

While we launched new features for Looppanel, the adoption for them was quite poor.

To improve the discoverabilty of features like inviting Loop bot to zoom call, adding calls to projects, and selecting and saving transcript text, we made changes to the information architechture of the product.

Home page redesign

Restructuring navigation and making it easier to invite bot

I restructured the navigation bar to resemble the new structure of product

project>call>notes


At the call level we made it easier to go into a call or move a call to a project if it is was unassigned.

Old home page and new home page

Updated horizontal menu and copy notetaking link

Call/Editor screen redesign

In the older call/editor screen, there were 3 types of views to show text linked to the transcript.

  1. A video snippet created from selecting the transcript text and adding a tag.
  2. Moment bookmarked during note-taking.
  3. Note taken during note taking

Due to multiple views of the same transcript text, the screen looked inconsistent. Users were also confused with the purpose of each. The addition of question script would've added more complexity
